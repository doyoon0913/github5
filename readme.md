# 📌 파이썬 객체 지향 프로그래밍(OOP) 핵심 개념 총정리

이 문서는 파이썬의 핵심 패러다임인 **객체 지향 프로그래밍(Object-Oriented Programming)**의 기본 구조와 작동 원리를 직관적으로 이해하기 위해 정리한 가이드입니다.

---

## 1. 클래스(Class)와 인스턴스(Instance)

* **클래스 (Class)**
  * 현실 세계의 사물이나 개념을 컴퓨터가 이해할 수 있도록 만든 **'설계도' 또는 '틀'**입니다.
  * 어떤 데이터가 필요하고 어떤 기능이 있어야 하는지 정의만 해둔 상태입니다.
* **인스턴스 (Instance) / 객체 (Object)**
  * 클래스라는 설계도를 바탕으로 **메모리에 실제로 만들어낸 '실체'**입니다.
  * 하나의 설계도(클래스)로 서로 다른 고유한 값들을 가진 수많은 실체(인스턴스)를 찍어낼 수 있습니다.

---

## 2. 변수의 두 가지 종류 (공유 vs 독립)

클래스 내부에서 데이터를 담는 상자(변수)는 그 성격에 따라 두 가지로 나뉩니다.

### 👥 클래스 변수 (공유 메모리)
* **정의**: 클래스 무대 전체가 공통으로 사용하는 하나의 거대한 저장소입니다.
* **특징**: 이 설계도로 생성된 **모든 인스턴스가 공간을 공유**합니다. 
* **예시**: 학교의 `학년/반` 정보나 은행의 공통 `기본 이자율`처럼, 모든 객체에 똑같이 일괄 적용되어야 하는 데이터에 사용합니다. 클래스 변수를 수정하면 모든 인스턴스가 동시에 영향을 받습니다.

### 👤 인스턴스 변수 (독립 메모리)
* **정의**: 각 객체가 자기 주머니 속에 개별적으로 차고 있는 독립된 저장소입니다.
* **특징**: 파이썬에서는 고유한 방임을 나타내기 위해 변수명 앞에 반드시 **`self.`**을 붙여 선언합니다.
* **예시**: 학생마다 다른 `이름`과 `직책`, 계좌마다 다른 `잔액`처럼 객체 고유의 데이터를 저장할 때 사용하며, 서로의 데이터에 절대 간섭하지 않습니다.

---

## 3. 메서드(Method)와 `self` 의 본질

### 🛠️ 메서드 (Method)
* 클래스 설계도 내부에 정의된 **'함수(기능)'**를 의미합니다. 
* 은행 계좌 클래스에 있는 '입금하기', '이자 계산하기', 동물 클래스에 있는 '소리내기' 같은 행동을 정의합니다.

### 🎯 `self` (주인공 지목 주소)
* 파이썬 클래스 내의 모든 메서드에서 첫 번째 매개변수로 지정하는 약속된 단어입니다.
* `self`는 **"지금 이 기능을 실행하고 있는 바로 그 객체(자기 자신)"**를 가리키는 메모리 주소 역할을 합니다.
* **작동 원리**: 여러 계좌 중 'A의 계좌'에서 입금 메서드를 호출하면, 컴퓨터는 `self`를 보고 다른 계좌가 아닌 정확히 'A 계좌의 인스턴스 변수(잔액)'를 찾아가서 값을 올리게 됩니다.

---

## 4. 상속(Inheritance)과 오버라이딩(Overriding)



### 🧬 상속 (Inheritance)
* 이미 기존에 잘 만들어진 부모 클래스의 설계도(데이터와 기능)를 자식 클래스가 **그대로 물려받아 재사용**하는 기술입니다.
* 코드의 중복을 획기적으로 줄여주며, "자식 클래스는 부모 클래스의 한 종류이다"라는 관계를 형성합니다.
* **예시**: `동물` 클래스(부모)가 가진 '이름 저장' 능력을 `강아지` 클래스(자식)가 한 줄의 코드로 모두 물려받아 그대로 사용할 수 있습니다.

### 🔄 메서드 오버라이딩 (Method Overriding)
* 부모 클래스로부터 물려받은 기능이 자식 클래스에 맞지 않을 때, **자식 입맛에 맞게 기능을 재정의(덮어쓰기)** 하는 것입니다.
* 부모가 정의한 함수와 **정확히 똑같은 이름**으로 자식 클래스에서 함수를 다시 만들면 부모의 기능은 가려지고 자식의 기능이 우선 실행됩니다.
* **예시**: 부모인 `동물` 클래스가 "동물이 소리를 냅니다"라고 웅얼거렸다면, 자식인 `강아지` 클래스는 이 기능을 덮어써서 "강아지가 멍멍 하고 웁니다"라는 구체적인 행동을 하도록 교체할 수 있습니다.
